## High-Priority Findings

### 1. `make test` does not build or reliably run e2e tests

Location:

- `Makefile:39-50`
- `README.md:77-79`

The README says `make test` runs all unit and end-to-end tests, but the Makefile only runs e2e tests if a previously-built `ltxpng` binary happens to exist:

```make
@if [ -f tests/e2e.sh ] && [ -f ltxpng ]; then \
  echo "=== End-to-end tests ==="; \
  tests/e2e.sh ./ltxpng || exit 1; \
  echo "=== All e2e tests passed ==="; \
fi
```

On a clean checkout, this succeeds while running only unit tests:

```sh
make clean && make test
```

This is a release-quality issue because CI could pass while never exercising the actual CLI.

Recommendation:

```make
test: ltxpng $(TEST_BIN)
```

Or split test targets explicitly:

```make
unit: $(TEST_BIN)
e2e: ltxpng
test: unit e2e
```

If e2e dependencies are optional, skip them explicitly with a clear message rather than implicitly based on whether the binary happens to exist.

### 2. Documented install flow is broken

Location:

- `README.md:15-18`
- `Makefile`

The README documents:

```sh
sudo make install
```

but the Makefile has no `install` target. Running `make -n install` fails.

Recommendation:

Add standard `PREFIX`, `BINDIR`, `install`, and possibly `uninstall` targets:

```make
PREFIX ?= /usr/local
BINDIR ?= $(PREFIX)/bin

install: ltxpng
	install -d $(DESTDIR)$(BINDIR)
	install -m 0755 ltxpng $(DESTDIR)$(BINDIR)/ltxpng
```

### 3. `pdfcrop` is required but not checked

Location:

- `README.md:8`
- `src/main.c:82-84`
- `src/main.c:209-217`

The README lists `pdfcrop` as required, and full-document mode invokes it. However, startup checks only `xelatex` and `gs`:

```c
if (!check_tool("xelatex")) return 127;
if (!check_tool("gs")) return 127;
```

If `pdfcrop` is missing, full-document mode returns a generic code `3`, likely with little or no useful diagnostic, instead of the documented `127` for a missing required tool.

Recommendation:

After classifying the fragment, check `pdfcrop` when `kind == FRAG_FULLDOC`, or check it unconditionally if it is considered a hard dependency.

---

### 4. Full-document margins do not honor TeX dimensions

Location:

- `README.md:34`
- `src/template.c:35`
- `src/main.c:196-206`

For wrapped-fragment mode, `margin` is passed to LaTeX as a TeX dimension:

```c
sb_appendf(buf, "\\documentclass[border=%s,varwidth]{standalone}\n", margin);
```

But full-document mode strips only the numeric prefix before passing the value to `pdfcrop`:

```c
const char *m = o.margin;
char *end;
double margin_val = strtod(m, &end);
if (end && *end) {
  snprintf(margin_bp, sizeof(margin_bp), "%.0f", margin_val);
} else {
  snprintf(margin_bp, sizeof(margin_bp), "%.0f", margin_val);
}
```

Examples:

- `-m 1in` becomes `1` bp, not approximately `72` bp.
- `-m 0.5in` becomes `0` bp because of `%.0f`.
- `-m 2pt` appears reasonable only by accident.

This violates the README claim that `-m, --margin DIM` accepts a TeX dimension.

Recommendation:

Either implement dimension parsing/conversion to bp for `pdfcrop`, or restrict and validate the accepted full-document margin format separately.

### 5. `--keep` leaks failed temp directories without telling the user where they are

Location:

- `src/main.c:276-281`

Current cleanup behavior:

```c
if (o.keep && exit_code == 0) {
  printf("%s\n", tmpdir);
} else if (!o.keep) {
  tmpdir_remove(tmpdir);
}
```

When `--keep` is used and the command fails, the temp directory is preserved but not printed. This leaks temporary state while also failing to provide the debugging information the user likely requested.

Recommendation:

Print the temp directory whenever `--keep` is set, using `stderr` on failure:

```c
if (o.keep) {
  fprintf(exit_code == 0 ? stdout : stderr, "%s\n", tmpdir);
} else {
  tmpdir_remove(tmpdir);
}
```

## Medium-Priority Findings

### 6. Output move/copy fallback can leave corrupted destination files

Location:

- `src/main.c:54-70`

`move_file()` falls back to copying directly into the destination using `fopen(dst, "wb")`.

Issues:

- Existing destination is truncated before a successful copy is guaranteed.
- `fread()` errors are not checked.
- `fflush()` / `fclose()` write errors are not checked.
- The temp source is unlinked even if close-time errors occur.
- No atomic replacement is used for cross-device fallback.

Recommendation:

For cross-device fallback, copy to a temporary file in the destination directory, check all read/write/close results, then `rename()` the temporary file into place.

### 7. Allocation failures are mostly unchecked

Locations include:

- `src/util.c:10-12`
- `src/util.c:20-23`
- `src/util.c:69-72`
- `src/run.c:123-126`
- `src/args.c:42`
- `src/args.c:67-68`

Examples:

- `sb_init()` dereferences the result of `malloc()` immediately.
- `sb_append()` assigns `realloc()` directly to the live buffer pointer.
- `read_all()` assigns `realloc()` directly.
- `extract_log_errors()` uses unchecked `realloc()` and `strdup()`.
- `parse_args()` does not check `calloc()` and silently ignores failed `strdup()`.

For a small CLI, exiting on out-of-memory may be acceptable, but it should be deliberate and safe.

Recommendation:

Add `xmalloc` / `xrealloc` helpers, or make `sb_*` functions return status and propagate allocation failures.

---

### 8. Argument parser silently ignores extra positional arguments

Location:

- `src/args.c:90-95`

The parser accepts only the first positional fragment and ignores the rest.

Example:

```sh
ltxpng a b c
```

This uses `a` and silently drops `b c`.

Given the documented usage `ltxpng [options] [FRAGMENT]`, extra positional arguments should be a usage error.

Recommendation:

```c
if (optind < argc) {
  o->fragment = argv[optind++];
  if (strcmp(o->fragment, "-") == 0)
    o->fragment = NULL;
}

if (optind < argc) {
  fprintf(stderr, "error: too many positional arguments\n");
  free(preambles);
  return 1;
}
```

Take care to free duplicated preamble strings on error as well.

---

### 9. DPI parsing can overflow `int`

Location:

- `src/args.c:49-57`

The parser uses `strtol()` and casts the result to `int` without checking `INT_MAX` or `ERANGE`.

A very large `--dpi` can wrap into an invalid value before being passed to Ghostscript.

Recommendation:

Include `<limits.h>` and `<errno.h>`, reset and check `errno`, and require `val <= INT_MAX`.

---

### 10. Recursive temp cleanup follows directory symlinks

Location:

- `src/run.c:63-67`

`remove_recursive()` uses `stat()`, which follows symlinks:

```c
if (stat(full, &st) == 0 && S_ISDIR(st.st_mode))
  remove_recursive(full);
else
  unlink(full);
```

If a symlink to a directory appears inside the temp directory, cleanup recurses into the symlink target and can delete files outside the temp tree.

The temp directory is private, so this may not be easily exploitable in normal usage, but cleanup code should still use `lstat()` and unlink symlinks rather than follow them.

Recommendation:

Use `lstat()` and recurse only into real directories that are not symlinks.

---

### 11. `find_on_path()` does not match real executable lookup semantics

Location:

- `src/run.c:79-109`

The implementation checks only `S_IXUSR`:

```c
if (stat(full, &st) == 0 && (st.st_mode & S_IXUSR)) {
```

Issues:

- Ignores group/other execute bits.
- Does not use `access(path, X_OK)`.
- Does not verify the path is a regular file.
- Skips empty `PATH` components, which conventionally mean the current directory.
- Performs a separate lookup before later using `execvp()`, so there is still a time-of-check/time-of-use window.

Recommendation:

Either remove eager checks and rely on `execvp()` returning `127`, or implement lookup with `access(X_OK)` and use resolved absolute paths consistently.

## Lower-Priority Findings

### 12. Empty stdin is accepted despite code implying it is an error

Location:

- `src/main.c:97-98`

`read_all()` returns an allocated empty string for empty input. The code checks only for `NULL`:

```c
fragment = read_all(stdin, &len);
if (!fragment) { fprintf(stderr, "error: empty stdin\n"); return 1; }
```

This branch catches allocation failure, not empty stdin.

Recommendation:

Decide whether empty input is valid. If not, check `len == 0`, and also consider checking after `chomp()` if whitespace-only input should be rejected.

---

### 13. `make debug` does not force a debug rebuild

Location:

- `Makefile:19-20`

`make debug` adds sanitizer/debug flags, but if `ltxpng` was already built, `make debug` can report that there is nothing to do.

Recommendation:

Build debug artifacts separately, force a rebuild, or use a separate target such as `ltxpng-debug`.

---

### 14. Tests compile with many warnings

Locations include:

- `tests/test.h`
- `tests/test_template.c`

The production source builds cleanly, but tests emit many warnings under the repository's own `-Wall -Wextra -Wpedantic` flags.

Notable examples:

- Unused `pass` parameter from the `TEST` macro.
- Empty array initializers in `tests/test_template.c`, which are not C11.

This would block adopting `-Werror` later.

Recommendation:

Mark unused parameters explicitly in the test macro or harness, and replace empty arrays with `NULL` or non-empty fixtures where count is zero.

---

### 15. Help/version output goes to stderr

Location:

- `src/args.c:76-82`

Successful `--help` and `--version` output currently goes to `stderr`.

Recommendation:

Use `stdout` for successful help/version output. Reserve `stderr` for errors.

## Positive Notes

- The module split is clean for a first commit: `args`, `detect`, `template`, `run`, and `util` are understandable.
- Unit coverage is solid for the current scope.
- E2E coverage is valuable and already tests real LaTeX/Ghostscript behavior.
- Subprocess execution uses `execvp()` with argv arrays rather than shell interpolation, avoiding a large class of shell-injection bugs.
- Temp directories use `mkdtemp()`, which is the right primitive.
- Generated build artifacts are ignored and not tracked.
